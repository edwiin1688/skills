# 日誌哨兵

## 描述
此技能快速從 Docker 容器或系統服務檢索和過濾日誌，以識別錯誤、警告和例外。它過濾掉雜訊（INFO/DEBUG）以專注於問題。

## 使用方式
當使用者回報執行時錯誤、「500 錯誤」、要求「檢查日誌」或使用 `/scan-logs` 時使用此技能。

## 工作流程
1.  **識別目標**：決定要檢查哪個服務/容器（例如：`grafana`、`app_server`）。如果未知，列出運行中的容器。
2.  **擷取日誌**：執行 `docker logs --tail 200 <container_id>`（或類似指令）。
3.  **過濾**：
    -   保留包含：`ERROR`、`WARN`、`Exception`、`Fatal`、`Fail` 的行。
    -   捨棄一般的 `INFO`、`DEBUG` 行，除非它們提供錯誤所暗示的關鍵上下文。
4.  **報告**：
    -   呈現過濾後的日誌行。
    -   如果找到堆疊追蹤或特定錯誤檔案/行，請盡可能使用 `glance_file` 或 `view_file` 指向原始碼檔案。

## 指令
### `/scan-logs [service_name]`
掃描指定服務的日誌。

## 報告
在解釋日誌或向使用者報告發現時，您**必須**使用**繁體中文**。
