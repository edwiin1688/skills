# Code Reviewer 技能說明

此技能將 AI 設定為一名**專業的程式碼審核專家 (Expert Code Reviewer)**，專注於提升程式碼品質、安全性與效能。

## 核心功能與作用

此技能遵循標準化且嚴謹的審查流程，確保程式碼變更符合最佳實踐：

1.  **範疇分析 (Analyze)**：審視暫存的變更或特定檔案。確保變更範疇合理，且僅包含解決問題所需的最少改動。
2.  **風格規範 (Style)**：確保程式碼遵循專案既定的慣例與慣用法（特別是 `GEMINI.md` 中定義的規範）。
3.  **安全性檢查 (Security)**：識別並標記任何潛在的安全漏洞。
4.  **測試驗證 (Tests)**：檢查新邏輯是否具備對應的測試覆蓋率，並確認測試是否能有效驗證變更。

## 使用方法

### 觸發條件
在對話中提及以下關鍵字或執行相關請求時，將自動觸發或建議使用此技能：
- 「回饋 (Feedback)」
- 「評論 (Review)」
- 「檢查 (Check)」

### 適用場景
- 當你完成一段功能實作，需要確認是否符合專案風格時。
- 在準備提交（Commit）或發起合併請求（PR）前，進行自我檢查。
- 需要針對安全性或變更範疇進行第三方檢視時。

## 輸出規範

審查意見將以簡潔的條列方式呈現，並分為兩個部分：
- **優點 (Strengths)**：肯定程式碼各方面的優點。
- **改進機會 (Opportunities)**：指出具體的缺失並提供修正方向。

---
*註：與 `reviewer` (極致嚴苛版) 不同，此技能採用專業且建設性的口吻，而非批判性風格。*
