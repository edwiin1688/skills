# 語義化 Git

## 描述
此技能協助確保所有 Git 提交都嚴格遵循在 `GEMINI.md` 中定義的專案慣用提交格式。它分析已暫存的變更並生成符合規範的提交訊息。

## 使用方式
當使用者準備提交變更、要求提交訊息或使用指令 `/commit-gen` 時使用此技能。

## 工作流程
1.  **檢查狀態**：執行 `git status` 查看已暫存的內容。
    -   如果沒有暫存任何內容，警告使用者。
2.  **分析差異**：執行 `git diff --staged` 了解變更內容。
3.  **生成訊息**：建構訊息的邏輯：
    -   **格式**：`<type>(<scope>): <subject>`
    -   **類型**：根據差異選擇 `feat`、`fix`、`docs`、`style`、`refactor`、`perf`、`test`、`chore` 或 `revert`。
    -   **範圍**：識別受影響的模組或檔案。
    -   **主旨**：以**繁體中文**撰寫簡潔的摘要。
    -   **內文**：以**繁體中文**詳細說明，解釋變更的*原因*和*內容*。
4.  **輸出**：呈現 `git commit -m "..."` 指令供使用者審查和批准。

## 指令
### `/commit-gen`
為目前暫存的變更生成提交訊息。

## 報告
在提案提交訊息或向使用者解釋變更時，您**必須**使用**繁體中文**。
